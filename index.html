About Ariadne

Main Scanning Features

Scan Site: Initiates a comprehensive scan to find all pages of a website. It uses a three-stage process: first, it looks for a sitemap.xml file. If not found, it launches an automated browser to crawl the site by following hyperlinks, simulating user scrolling to find dynamically loaded content. Finally, it analyzes the site's JavaScript files to discover hidden paths.

Find Subdomains: Discovers subdomains associated with the main domain using several advanced techniques. It aggregates results from passive sources like Google, Certificate Transparency logs, and ThreatCrowd, and then validates each result to ensure it is live. For deeper scans, it can also perform an active brute-force attack using a large wordlist to find unlisted subdomains.

Scan Options

Depth: Controls how many 'layers' deep the 'Scan Site' crawler will go. A depth of 1 scans the starting page. A depth of 2 scans that page and every page it links to. Higher depths can significantly increase scan time.

Follow External Links: When checked, this allows the 'Scan Site' crawler to follow hyperlinks to different websites, up to the specified depth.

Enable Brute-Force: When finding subdomains, checking this box enables the active brute-force scan, which is more thorough but takes longer to complete.

Results & Actions

Results List: Displays all pages or subdomains found. You can select specific items using the checkboxes.

Export/Archive Selected: Allows you to save the selected items to a CSV file or submit them to be permanently saved on the Internet Archive for preservation. If no results are found during a scan, you can still archive the base URL.

Individual Actions (Diff, Tech, Headers, Scan): Buttons next to each result allow you to perform actions on that specific item, such as initiating a new scan on a subdomain, running a detailed analysis, or checking for changes.

Change Detection

Check for Changes (Diff): This powerful feature compares the live version of a page against its most recent snapshot on the Internet Archive. It queries the archive for the latest successful capture, fetches both versions, and displays a side-by-side comparison highlighting any text that has been added or removed. If no changes are found, it will inform you of the date of the last successful archive. If changes are detected, you have the option to archive the new version directly.

Visual Maps

View Map: After a scan, this button generates an interactive visual sitemap or subdomain map. Nodes on the map are clickable and will open the corresponding URL in a new tab.

Map Legend: Provides key information about the scanned host, including its IP address, domain provider, the number of items found, and the total scan time.

Analysis Tools

Detect Tech: Analyzes a URL to identify the web technologies it is built with (e.g., WordPress, React).

Analyze Headers: Inspects a URL's HTTP response to check for important security headers.

Check All For Broken Links: Tests every link found during a site scan to see if it leads to a "404 Not Found" error.

Automated Scanning

Scan Scheduler: This feature transforms the tool into an automated monitoring system. You can schedule recurring jobs to automatically perform a subdomain search, run a diff check on each result, and archive any subdomains that have changed. Schedules can be set to run daily, weekly, bi-weekly, monthly, or every other month, and can be managed from the Scheduler modal.

Configuration

Proxy Settings: Allows you to provide a list of proxy servers. When archiving pages, the tool will rotate through this list to find a working proxy. You can also enable or disable the use of proxies entirely with a toggle switch.
