About Ariadne

Main Scanning Features

Scan Site: Initiates a comprehensive scan to find all pages of a website. It uses a three-stage process: first, it looks for a sitemap.xml file. If not found, it launches an automated browser to crawl the site by following hyperlinks, simulating user scrolling to find dynamically loaded content. Finally, it analyzes the site's JavaScript files to discover hidden paths.

Find Subdomains: Discovers subdomains associated with the main domain using several advanced techniques, including checking Certificate Transparency logs, performing various DNS lookups (MX, NS), and using Reverse DNS lookups.

Scan Options

Depth: Controls how many 'layers' deep the 'Scan Site' crawler will go. A depth of 1 scans the starting page. A depth of 2 scans that page and every page it links to. Higher depths can significantly increase scan time.

Follow External Links: When checked, this allows the 'Scan Site' crawler to follow hyperlinks to different websites, up to the specified depth.

Results & Actions

Results List: Displays all pages or subdomains found. You can select specific items using the checkboxes.

Export/Archive Selected: Allows you to save the selected items to a CSV file. You can also submit pages to be permanently saved on the Internet Archive for preservation.

Individual Actions (Scan, Tech, Headers): Buttons next to each result allow you to perform actions on that specific item, such as initiating a new scan on a subdomain or running a detailed analysis.

Visual Maps

View Map: After a scan, this button generates an interactive visual sitemap or subdomain map. Nodes on the map are clickable and will open the corresponding URL in a new tab.

Map Legend: Provides key information about the scanned host, including its IP address, domain provider, the number of items found, and the total scan time.

Analysis Tools

Detect Tech: Analyzes a URL to identify the web technologies it is built with (e.g., WordPress, React).

Analyze Headers: Inspects a URL's HTTP response to check for important security headers.

Check All For Broken Links: Tests every link found during a site scan to see if it leads to a "404 Not Found" error.

Configuration

Proxy Settings: Allows you to provide a list of proxy servers. When archiving pages, the tool will rotate through this list to avoid being rate-limited by the Internet Archive.
